Documentation front-end

Pour la documentation back-end, c'est là bas que ça se passe.

Etat de la documentation

La plupart des pages ne sont pas encore disponible, néanmoins les icônes associées aux liens vous permettent de connaître l'état de chacune des pages.

  • Les pages marquées sont terminées ;
  • Les pages marquées sont en cours de rédaction ;
  • Les autres pages ne sont pas encore commencées.

Participer à la rédaction de la documentation

Vous pouvez rédiger ou corriger la documentation directement sur Github ou via les liens situés en haut à droite des différentes pages de la documentation. Vous serez directement amenés vers la page d'édition sur Github qui créera un fork pour vous, si vous validez une Pull Request sera soumise, l'équipe se chargera alors de vérifier et de merge votre contribution.

Style de la documentation

Si le style de la documentation ressemble au site c'est qu'il se base quasi-entièrement sur le style utilisé par Zeste de Savoir en production (il y a quelques ajouts pour les besoins dédiés à la documentation).

Le fait que ce soit la version en production qui soit chargée peut entraîner des incohérences entre ce que vous voyez en développement et ce qui est présenté sur cette documentation.
